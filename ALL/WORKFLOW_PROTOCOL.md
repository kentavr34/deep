### 2. ПРОТОКОЛ НАПИСАНИЯ ГЛАВЫ (ШАГИ)

**Перед тем как написать главу, я обязан:**

#### Шаг 1. Прочитать предыдущую главу
- Открыть последнюю написанную главу в `CHAPTERS/BOOK_X/`
- Прочитать её полностью
- Зафиксировать, на чём она закончилась

#### Шаг 2. Прочитать тезисы текущей главы
- Открыть `BOOK_HEADING_THESES.md`
- Найти главу, которую предстоит писать
- Прочитать тезисы — что должно произойти

#### Шаг 3. Прочитать тезисы следующих глав
- Прочитать тезисы 2–3 следующих глав
- Понять, куда ведёт сюжет
- Убедиться, что текущая глава логично подводит к следующей

#### Шаг 4. Расположить сцены во временном интервале
- Определить, сколько времени прошло между предыдущей главой и текущей
- Определить, сколько времени займёт текущая глава
- Убедиться, что хронология не нарушена

#### Шаг 5. Написать главу
- Учитывая прочитанное, создать текст
- Соблюдать стиль (`STYLE_CODEX_V2.md`)
- Соблюдать канон (`WORLD_BIBLE_MASTER.md`)
- Следить за целевым объёмом (см. раздел 3)

#### Шаг 6. Чекап (обязательно!)
Проверить по чек-листу (см. раздел 4)

#### Шаг 7. Если чекап пройден — отправить автору
#### Шаг 8. Если чекап не пройден — исправить несоответствия, затем повторить чекап

### 3. ПРОТОКОЛ ПОДСЧЁТА ОБЪЁМА (ОБЯЗАТЕЛЬНЫЙ)

**Формула:**
- **1 страница печатного текста = 2500 знаков с пробелами**
- Это стандарт книжного формата (шрифт 11 pt, интервал 1.2, поля 2 см)

**Перед написанием главы:**
1. [ ] Зафиксировать **целевой объём в знаках** (целевые страницы × 2500)
2. [ ] Записать в `NOTES/CHAPTER_STATS.md`

**После написания главы:**
1. [ ] Подсчитать **реальное количество знаков с пробелами**
   - Использовать функцию статистики в текстовом редакторе
   - Или онлайн-инструмент (anycount.com, text.ru)
2. [ ] Разделить на 2500 — получить **реальное количество страниц**
3. [ ] Сравнить с целевым объёмом
4. [ ] Если отклонение >10% — добавить или сократить текст
5. [ ] Зафиксировать результат в `NOTES/CHAPTER_STATS.md`

**После завершения книги:**
1. [ ] Просуммировать знаки всех глав
2. [ ] Разделить на 2500 — получить **реальный объём книги в страницах**
3. [ ] Сравнить с целевым объёмом книги
4. [ ] Зафиксировать в `NOTES/BOOK_STATS.md`

**Запрещено:**
- Предоставлять данные о количестве страниц без реального подсчёта знаков
- Использовать оценочные цифры («примерно», «около», «~» без расчёта)
- Умножать знаки на 2, принимая 5000 знаков за страницу

### 4. ЧЕК-ЛИСТ ПЕРЕД ОТПРАВКОЙ ГЛАВЫ

- [ ] **Связь с предыдущей главой:** начинается ли глава с того места, где закончилась предыдущая?
- [ ] **Соответствие тезисам:** все ли сцены из `BOOK_HEADING_THESES.md` присутствуют?
- [ ] **Хронология:** не нарушена ли последовательность событий?
- [ ] **Стиль:** первое лицо? внутренние монологи? диалоги с подтекстом?
- [ ] **Канон:** нет запрещённых имён (Жасмин, Кен, медальон)?
- [ ] **Объём:** подсчитаны знаки? соответствует целевому? (отклонение не более 10%)
- [ ] **Нет лишних маркеров:** в конце главы только чистый текст

---

### 5. ЗАПРЕТЫ (НАРУШЕНИЯ, КОТОРЫЕ Я НЕ ДОЛЖЕН ДОПУСКАТЬ)

1. **Не писать главу, не прочитав предыдущую.**
2. **Не писать главу, не прочитав тезисы текущей и следующих глав.**
3. **Не отправлять главу без чекапа.**
4. **Не менять название главы без обновления структуры.**
5. **Не начинать главу не с того места, где закончилась предыдущая.**
6. **Не добавлять сцены, которых нет в тезисах, без согласования.**
7. **Не предоставлять данные о количестве страниц без реального подсчёта знаков.**

### 6. ЕСЛИ Я ОШИБСЯ

1. **Признать ошибку сразу** — без оправданий.
2. **Указать, в чём ошибка** — конкретно, по пунктам.
3. **Предложить исправление** — конкретный план действий.
4. **После исправления — повторить чекап.**
